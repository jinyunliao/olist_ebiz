# Contributing

Thank you for considering contributing to this project. To keep the project professional and client-ready, please follow these guidelines:

- Open an issue to discuss major changes before submitting a PR.
- Create small, focused pull requests with descriptive titles and linked issues.
- Run `make lint` / `sqlfluff` on SQL changes and `make check` to validate dbt artifacts.
- Include tests and docs updates for any behavioral changes.
- Follow the repository style for SQL and Python.

Maintainers will review PRs and provide constructive feedback.
